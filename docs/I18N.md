# QuantEdge — 国际化（i18n）规范

三档语言：**简体中文 `zh-CN` / 繁体中文 `zh-TW` / English `en`**。
所有实现集中在 [`frontend/src/i18n.jsx`](../frontend/src/i18n.jsx)，无第三方 i18n 框架。

---

## 1. 核心机制

**简体中文作 key 穿透**。每条 UI 文案以简体中文原文作为字典 key：

| 语言 | `t('量化评分')` 行为 |
|------|------|
| `zh-CN` | 原文穿透 → `量化评分` |
| `zh-TW` | 查 `TW` 手工字典；未命中走 **opencc `s2twp`** 自动转 → `量化評分` |
| `en` | 查 `EN` 字典 → `Quant Scoring`；未命中穿透原文 |

- **EN 字典**：~1850 条手工翻译（`const EN = {...}`）。
- **TW 字典**：仅放 opencc 兜底不理想、需统一品牌/术语的少数词。绝大多数繁体由 opencc 自动得到（含台湾用语：軟體 / 資訊 / 伺服器 / 最佳化 / 登入 等）。
- **opencc 误转修正**：`TW_POSTFIX` 对 opencc 输出做二次替换，目前 3 条：
  - `程式碼 → 代碼`（股票代码语境）
  - `許可權 → 權限`（通知/IT 语境台湾标准）
  - `覆盤 → 復盤`（金融「复盘 review」的「复」是「重复」义）

切换语言：顶栏 `LangSwitcher`（简/繁/EN）或用户面板偏好行，写入 `localStorage['quantedge_lang']`。旧值 `'zh'` 自动迁移为 `'zh-CN'`。

## 2. 给开发者：加新文案的规范

### 2.1 React 组件里
```jsx
import { useLang } from "../i18n.jsx";        // 路径按层级调整

function MyCard() {
  const { t, lang } = useLang();
  return <span>{t('持仓信息')}</span>;          // ✅ 简体作 key
}
```
- **永远不要**在 JSX 里写裸中文字面量 `<span>持仓信息</span>` —— 会被 `audit:i18n` 拦截。
- 带参数：`t('{n}个标的', { n: 543 })` → `543个标的` / `543 securities`。
- **属性也要包**：`title={t('刷新')}`、`aria-label={t('关闭')}`、`placeholder={t('搜索...')}`。

### 2.2 加英文翻译
在 `i18n.jsx` 的 `EN` 字典加一条：`'持仓信息': 'Position Info',`
- 不加 → 英文模式穿透显示中文（不会报错，但不完整）。
- 繁体**无需**手动加 —— opencc 自动转；只有发现误转时才补 `TW_POSTFIX` 或 `TW` 字典。

### 2.3 非 React 纯函数场景
用 `tStatic(text, lang)`（无 hook），如 `displayTicker`。

### 2.4 数据驱动的中文（公司名 / 板块 / 描述）
- 中文公司名：`isZh(lang) ? t(s.nameCN || STOCK_CN_NAMES[s.ticker] || s.name) : enFallback(s.name, s.ticker)`
- `enFallback(name, ticker)`：英文模式下中文名 → 查 EN 字典 → 命中用英文、未命中用 ticker 兜底，避免英文界面出现中文。
- `localeFor(lang)`：给 `toLocaleString` / `Intl` 用，输出 `zh-CN` / `zh-TW` / `en-US`。**不要**硬编码 `"zh-CN"`。

### 2.5 变量名冲突
组件里已有 `const { t } = useLang()` 时，注意 `.map((t) => ...)` 等会 shadow。重命名 map 参数为 `tr` / `tk` / `tg`。

## 3. 防回归工具

### `npm run audit:i18n`
扫 `pages/` + `components/` + `quant-platform.jsx` 的 JSX 文本节点，找没包 `t()` 的 hardcoded 简体。
- 默认非阻塞退出 0（lint 提醒）；`--strict` 退出 1（可挂 CI）。
- 当前状态：**0 处**（全部已 i18n）。
- 已知盲区（脚本不报，繁体经 opencc 正常转，仅英文穿透）：`title`/`aria` 属性、动态拼装长句（如 CharacterProfile `paragraph`）。

### `src/i18n.test.jsx`（33 例）
覆盖纯函数（`localeFor` / `isZh` / `hasCJK` / `enFallback` / `tStatic`）+ `LangProvider`/`useLang` 行为 + opencc 三条 post-process 修正断言。改 i18n 后必跑。
